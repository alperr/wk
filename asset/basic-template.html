this page is generated by a command line interface: wk

web apps generated with wk are composed of components

what is a component?

  • every independently functioning part of a user interface is called as component
  • every component contains one javascript, css and html file
  • components are stored under src/components/ as individual folders
  • every project contains a main component named app
  • you can tweak this app component's style by editing src/components/app/app.css file
  • markup of web-app component is written into src/components/app/app.html
  • components are defined as functions in .js files and any component can create another one just by calling the constructor of that component
  • program entry point is src/components/web-app/web-app.js

all other .js files should be placed under src/ folder. wk concatanates every .js files in src folder (including javascript files of each component) and serves them as a single javascript file, hence you don't have to use require() or import to develop in a multiple file javascript environment, everything you write is registered to global.

run wk help to learn details of CLI
